Creating Your First App

Welcome to the guide on creating your first app on the Cortic Platform. Follow the steps below to ensure a smooth development experience.

1. Setting up Visual Studio Code (VSCode)

Begin by ensuring you have the latest version of VSCode installed on your machine.

  1. Download and install VSCode from the official website if you haven't already.
  2. Open VSCode and navigate to the extensions section. Ensure you have the "code command" extension installed.
  3. After installation, ensure that you add the "code" command to your system's PATH.
<screenshot>

2. Create a New App

From the Dev Portal, select the option to create a new app. Follow the prompts to name and configure your app's initial settings.

<screenshot>

3. Understanding the App Directory Structure

The app directory is structured in a specific way to facilitate seamless development. Here is a brief overview:

  • Manifest File: This file contains metadata about your app. You will need to edit this to provide specifics about your application.
  • Icons: Your apps default icon is in the asset folder. You may replace it with your own icon. Your icon should be 100px by 100px in .png format.
  • Source Code: This is where the main logic of your app resides. You app should inherit from the App class and implement several methods: setup, process, teardown, and on_exception.

4. Importing the face_landmarks Service

Head over to the Store and search for the "face_landmarks" service. Follow the instructions to import it into your app.

<screenshot>

5. Inspecting the service_registry.py File

Once you've imported the service, you can find its autogenerated wrapper function by inspecting the service_registry.py file. Note down the function name as you'll use it in your app.

<screenshot>

6. Running the App and Inspecting Logs

Run your app using the provided execution command. To troubleshoot or get insights into the application's performance, open the console window and inspect the logs.

<screenshot>

7. Checking the Device Page

Visit the device page to view the current status of the face_landmarks service. This gives a real-time update on its performance and any detected landmarks.
