class cortic_platform.sdk.ui.basic_widgets.Container(rect=[0, 0, 0, 0])¶

Bases: Widget

A Container widget is the basis of an App’s UI construction, any other widgets must be added to a Container widget’s children variable for display.

__init__(rect=[0, 0, 0, 0])¶

Initializes the Container widget.

Parameters:

rect (list) – The size and starting position of the widget. [0, 0, 100, 100] represents a widget with a width of 100 and a height of 100, and starting at the top left corner of the screen.

Variables:

  • alignment (str) – Determines the alignment of the widgets in this Container.

  • child_fitting_mode (str) – Determines how the widgets in this Container are fitted. This is useful when the widgets in this Container are larger than the Container itself. The options are “fit” and “full_size”. “fit” means the widgets will be scaled down to fit the Container, while “full_size” means the widgets will be displayed in their original size, and the Container will be scrollable.

Example:

from cortic_platform.sdk.ui.basic_widgets import Container
container = Container([0, 0, 100, 100])

remove_child(widget)¶
Remove a widget from the Container’s children list. If the widget is not a Widget object or

the widget is not in the Container’s children list, no action will be taken.

Parameters:

widget (Widget) – The widget to be removed.

Example:

from cortic_platform.sdk.ui.basic_widgets import Container, Button
container = Container([0, 0, 100, 100])
button = Button([0, 0, 100, 100])
container.add_child(button)
container.remove_child(button)

Remove all widgets from the Container’s children list.

class cortic_platform.sdk.ui.widget.Widget(rect=[0, 0, 0, 0])¶

Bases: object

This class represent a generic widget used in the UserInterface object. Developers can extend this class to create custom widgets. They can customize the internal parameters of their widgets, and combine any pre-defined elements provided by the cortic_platform sdk to customize the widgets for displaying different types of app data.

Note: The widgets created using this API do not represent any actual renderable UI code. These widgets are just instructions for the display driver, which can convert these instructions into actual UI code for rendering on a local or remote client.

Variables:

  • widget_tree (WidgetTree) – A reference to the widget tree that this widget belongs to.

  • alias (str) – The alias of the widget. This is used to identify the widget in the widget tree.

  • canvas (Canvas) – The canvas of the widget. This is used to add additional elements to the widget.

  • rect (list) – The size and starting position of the widget. [0, 0, 100, 100] represents a widget with a width of 100 and a height of 100, and starting at the top left corner of the screen.

  • state (WidgetState) – The state of the widget.

  • alpha (int) – Transparency of this widget.

  • tooltip (str) – The tooltip of the widget.

  • corner_radius (int) – The radius of the widget’s corners.

  • use_custom_corner_radius (bool) – Controls whether the widget uses custom radius for its corners.

  • custom_corner_radius (list) – The custom radius of the widget’s corners. The list should have 4 elements, each element represents the radius of a corner. The order is [top_left, top_right, bottom_left, bottom_right].

  • opacity (int) – The opacity of the widget.

  • visible (bool) – Controls whether the Widget is included in the UserInterface’s widget tree or not.

  • visibility_animation_duration (int) – The duration of the visibility animation during show/hide operations. The unit is in milliseconds.

  • border_color (str) – The color of the border.

  • border_thickness (int) – The thickness of the border.

  • background_color (str) – The color of the background.

  • data (dict) – The data that the widget uses for display. Developers can set this data during runtime.

  • on_widget_event (function) – A custom event handler that the developer can provide if the widget supports event callbacks, e.g., Button.

  • capture_keyboard_event (bool) – Controls whether the Widget captures keyboard events.

  • capture_mouse_event (bool) – Controls whether the Widget captures mouse events.

  • on_keyboard_event (function) –

    A custom keyboard event handler that the developer can provide if the widget supports keyboard event callbacks. The handler should have the following signature: on_keyboard_event(event_type, event_data).

    Event_type is one of the following:

    • ”key_down”

    • ”key_up”

    Event_data is a dictionary containing the following keys:

    • ”key”: The key that was pressed or released.

    • ”repeat”: True if the key was held down for a long time, False otherwise.

  • on_mouse_event (function) –

    A custom mouse event handler that the developer can provide if the widget supports mouse event callbacks. The handler should have the following signature: on_mouse_event(event_type, event_data).

    Event_type is one of the following:

    • ”tap”

    • ”double_tap”

    • ”long_press_down”

    • ”long_press_move_update”

    • ”long_press_up”

    • ”secondary_tap”

    • ”secondary_double_tap”

    • ”secondary_long_press_down”

    • ”secondary_long_press_move_update”

    • ”secondary_long_press_up”

    • ”tertiary_long_press_down”

    • ”tertiary_long_press_move_update”

    • ”tertiary_long_press_up”

    • ”pan_start”,

    • ”pan_update”,

    • ”pan_end”,

    • ”wheel_scroll”

    Event_data is a dictionary containing the following keys:

    For event_type other than “wheel_scroll”, “pan_update”, and “pan_end”:

    • ”x”: The x coordinate of the mouse event.

    • ”y”: The y coordinate of the mouse event.

    For event_type “wheel_scroll”:

    • ”x”: The x coordinate of the mouse event.

    • ”y”: The y coordinate of the mouse event.

    • ”scroll_delta_x”: The x delta of the mouse wheel.

    • ”scroll_delta_y”: The y delta of the mouse wheel.

    For event_type “pan_end”:

    • ”x”: The x coordinate of the mouse event.

    • ”y”: The y coordinate of the mouse event.

    • ”velocity_x”: The x velocity of the mouse when the drag ends.

    • ”velocity_y”: The y velocity of the mouse when the drag ends.

    For event_type “pan_update”:

    • ”x”: The x coordinate of the mouse event.

    • ”y”: The y coordinate of the mouse event.

    • ”delta_x”: The x delta of the mouse when the drag ends.

    • ”delta_y”: The y delta of the mouse when the drag ends.

cortic_platform.sdk.ui.widget_tree module¶

class cortic_platform.sdk.ui.widget_tree.WidgetTree¶

Bases: object

A class that allows developers to create and customize the user interface for their apps. Every Cortic app includes a default WidgetTree object generated during app initialization. Developers can combine numerous widgets from the widget library and add them into the WidgetTree. The WidgetTree object then builds the user interface based on the widgets added into it.

Variables:

  • rendering_target (str) – The target device for the WidgetTree object. The default value is “”, meaning the WidgetTree object will be rendered on the device that is running the app. If the WidgetTree object is going to be rendered on a different device, the target device should be set to the device’s name.

  • scale_mode (str) – The scaling mode for the WidgetTree object. The default value is “aspect_fit”. The scaling mode can be set to either “aspect_fit” or “fit”.

  • enable_title_bar (bool) – A boolean value that indicates whether the WidgetTree object should display a title bar. The default value is True.

__init__()¶

Initializes a WidgetTree object. The default object contains an empty widget list. As deveopers add more and more widgets into it, the list will eventually contain a list of Container widgets, in which each Container widget can have none or many children widgets that came from the cortic_platform’s widget library.

add_child(widget)¶

This function accepts a Container Widget object and append it into its widget list. If user wants to add a Widget that is not a Container, they should make sure the Widget’s parent is a Container, and they should add this parent Container as child instead. If the child is still not a Container Widget, an error code will be returned.

Parameters:

widget (Container) – The Container Widget object that is going to be added into the WidgetTree object.

Returns:

A code that indicates whether the widget is successfully added into the WidgetTree object. A code of 0 means the widget is successfully added into the WidgetTree object. A non-zero code means the widget is not successfully added into the WidgetTree object.

Return type:

int

Example:

from cortic_platform import WidgetTree, Container
container = Container([0, 0, 100, 100])
self.ui.add(container)
remove_child(widget)¶

This function removes the given widget from the WidgetTree’s children list. If the widget is not in the WidgetTree object, an error code will be returned.

Parameters:

widget (Widget) – The widget that is going to be removed from the WidgetTree object.

Returns:

A code that indicates whether the widget is successfully removed from the WidgetTree object. A code of 0 means the widget is successfully removed from the WidgetTree object. A non-zero code means the widget is not successfully removed from the WidgetTree object.

set_rendering_target(target)¶

This function sets the target device for the WidgetTree object.

Parameters:

target (str) – The target device for the WidgetTree object. The target can be set to any device that supports the required display driver for the WidgetTree object.

Example:

from cortic_platform import WidgetTree

build(prebuilt_widget=None)¶

This functions convert the WidgetTree into a set of instruction and sends it to the display driver for rendering. The display driver on the target device will then render the WidgetTree object.

Example:

from cortic_platform import WidgetTree
container = Container([0, 0, 100, 100])
self.ui.add(container)
self.ui.build()
update(widget=None, use_existing_data=False)¶

This functions checks if there are updates in the widgets’ data or appearance. If there are udpates, it will send these latest changes to the display driver on the target device. The display driver on the target device will then render the updated WidgetTree object. If the widget parameter is not None, it will only update the specific widget and its children. If the widget is not a Widget object or it is not in the WidgetTree object, an error code will be returned.

Parameters:

  • widget (Widget) – The widget to update. If None, it will update the entire WidgetTree.

  • use_existing_data – If True, it will use the existing data in the widget. If False, it will use the latest data in the widget.

Returns:

A code that indicates whether the widget is successfully updated. A code of 0 means the widget is successfully updated. A non-zero code means the widget is not successfully updated.

Example:

from cortic_platform import WidgetTree
container = Container([0, 0, 100, 100])
my_image = Image([0, 0, 100, 100])
container.children.append(my_image)
self.ui.add(container)
self.ui.build()
my_image.data = "https://www.google.com/images/branding/googlelogo/1x/googlelogo_color_272x92dp.png"
self.ui.update()
